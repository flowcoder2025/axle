# 05. 사업계획서

AXLE의 핵심 기능입니다. **RAG 기반 초안 생성(Engine A)**과 **양식 정밀 편집(Engine B)**, 이 두 엔진을 조합해 수준 높은 사업계획서를 빠르게 완성합니다.

> 📌 **UI 안정화 안내** — 사업계획서 전용 UI는 현재 개발·안정화 단계입니다. 이 장의 절차·버튼 위치·스크린샷은 정식 릴리스 이후 최신화됩니다. 현재도 프로젝트 상세 페이지의 AI 작업 탭을 통해 AI 작업을 트리거할 수 있습니다.

---

## 이 장에서 할 수 있는 것

- 고객사 자료로 사업계획서 초안을 AI가 자동 생성 (Engine A)
- 지정된 양식(K-Startup, 벤처, 창업도약 등)에 맞게 정밀 편집 (Engine B)
- AI 자가 평가 + 개선안 제시
- 서류 완비 여부 사전 검증
- 완성본 DOCX 다운로드 및 고객사 전달

---

## 1. 두 엔진의 차이

| 엔진 | 역할 | 사용 시점 |
|------|------|---------|
| **Engine A (RAG 초안)** | 고객사 자료를 검색해 자유 형식 초안 생성 | 처음 작성 시, 아이디어 정리 |
| **Engine B (정밀 편집)** | 특정 양식(공고 템플릿)의 항목별 분량·스타일 맞추기 | 제출 직전 양식화 |

일반적인 흐름은 **A로 초안 생성 → B로 양식 맞춤** 순서입니다.

---

## 2. 초안 생성 (Engine A)

### 전제 조건

다음이 준비되어 있어야 품질이 보장됩니다.

- ✅ 고객사 **마스터 프로필** 생성 완료 ([01장](./01-고객사-관리.md))
- ✅ 고객사 관련 **서류 5개 이상** 업로드 (재무제표·기술자료·기존 계획서 등) ([03장](./03-서류-관리.md))
- ✅ 최근 **미팅 전사** 1건 이상 ([04장](./04-미팅-전사.md))

### 골든 패스

1. 프로젝트(타입: BIZ_PLAN) 상세 페이지 → **[사업계획서]** 탭.
2. **[초안 생성]** 클릭.
3. 옵션 선택.
   - *목적* — 지원사업 신청 / 투자 IR / 내부 전략 등
   - *대상 지원사업* — 등록된 ProgramInfo 중 선택 (선택 시 해당 공고 요건 반영)
   - *톤* — 공식적 / 설득형 / 기술 중심
4. **[생성 시작]** → AiJob이 등록되고 보통 2~5분 내 완료.
5. 완료 시 알림이 오고, 탭 화면에 초안이 섹션별로 표시됩니다.

> _스크린샷 준비 중 — 초안 생성 UI는 현재 개발·안정화 중입니다._

### 초안 검토

- **소스 인용** — 각 문단 우측에 사용된 서류·미팅이 표시됩니다. 클릭하면 원본으로 이동.
- **섹션별 재생성** — 마음에 안 드는 섹션만 우측 ⟳ 버튼으로 다시 생성.
- **인라인 편집** — 직접 텍스트를 수정할 수 있습니다 (저장 시 버전 기록).

---

## 3. 양식 정밀 편집 (Engine B)

지원사업 공고에는 "항목명·분량·표 구조"가 정해져 있습니다. Engine B는 초안을 해당 양식에 맞춥니다.

### 골든 패스

1. 사업계획서 탭에서 **[양식 적용]** 클릭.
2. 양식 선택.
   - 등록된 양식(예: `K-Startup 2026 예비창업`, `창업도약 2026`, `벤처 재도전` 등)
   - 또는 HWPX/DOCX 업로드하여 **커스텀 양식** 추가
3. AI가 초안 각 섹션을 양식 항목에 매핑하고, 분량·문체를 조정합니다.
4. 결과를 항목별로 검토합니다.

> _스크린샷 준비 중 — 정밀 편집 UI는 현재 개발·안정화 중입니다._

💡 **팁** — 양식은 한번 등록하면 조직 전체에서 재사용됩니다. 공공기관 공고가 나올 때마다 관리자가 양식 라이브러리를 갱신합니다.

### HWPX 한글 양식

한글(HWPX) 양식은 업로드 후 AXLE가 내부적으로 편집하여 그대로 내보냅니다. 한컴오피스에서 열어 바로 제출 가능합니다.

---

## 4. AI 자가 평가

초안 또는 정밀 편집본에 대해 AI가 스스로 평가합니다.

1. **[평가]** 버튼 클릭.
2. 평가 기준 선택.
   - **일반 기준** — 구조·설득력·구체성·일관성
   - **양식 기준** — 공고 심사 기준(있는 경우)
3. 섹션별 점수(0~10)와 개선안이 표시됩니다.

> _스크린샷 준비 중 — 자가 평가 UI는 현재 개발·안정화 중입니다._

점수가 낮은 섹션은 **[개선 적용]**을 누르면 AI가 제안한 문장으로 대체됩니다. (원본은 버전 이력에 보존됩니다.)

---

## 5. 서류 완비 검증

제출 전 **누락된 첨부 서류**를 체크합니다.

1. **[검증]** 버튼 클릭.
2. AXLE가 공고 요건과 현재 고객사의 서류 목록을 대조해 누락 항목을 표시합니다.
   - 예: "법인등기부등본 없음", "재무제표 2023년 누락"
3. 누락 항목은 [03장 체크리스트](./03-서류-관리.md#5-체크리스트)에 자동 반영되어, 해결 여부를 추적할 수 있습니다.

---

## 6. 최종 파일 다운로드

1. **[DOCX 다운로드]** — 일반 양식(기본 제출용)
2. **[HWPX 다운로드]** — 한글 양식(선택된 경우)
3. **[고객사에 발송]** — 이메일로 파일 전달 + EmailLog에 기록

---

## 자주 묻는 질문

- **초안이 너무 일반적이에요.** → 마스터 프로필과 서류가 충분하지 않을 때 흔한 증상입니다. 고객사 자료를 추가 업로드하고 재생성하세요.
- **양식 항목에 분량을 초과해요.** → 해당 섹션 **[요약 재생성]**으로 분량을 줄여 재생성할 수 있습니다.
- **수정한 초안이 사라졌어요.** → **[버전 이력]** 탭에서 이전 버전을 복원할 수 있습니다.
- **영문으로 작성 가능한가요?** → 네, 생성 옵션에서 *언어 = English* 선택 가능합니다. (한/영 혼용 양식은 제한적)
