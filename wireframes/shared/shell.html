Shell 규칙: 사이드바 1개 + topbar 1개. 사이드바는 install한 모듈 × 사용자 권한이 교차하는 nav만 표시. 모든 모듈이 같은 디자인 토큰(DESIGN.md) 사용 — 시각적 일관성.

사이드바 동적 렌더링 룰

buildSidebar(org, user)

module-registry + ReBAC
Step 1

조직 install 모듈 조회
getInstalledModules(orgId)

Step 2

사용자 권한 필터
userPerms.canAccess(moduleId)

Step 3

모듈별 nav + 공통 + (관리자) 합쳐 렌더

예시

"FlowCoder Inc."가 컨설팅(M1) + HR(M2) + 콘텐츠(M3) install. 사용자 A는 컨설팅 readwrite + HR read만 → 사이드바: 컨설팅 11nav + HR 4nav(읽기만) + 공통.
