③ 데이터 모델

tenantOrgId scope

// 모든 multi-org 적용 모듈의 테이블에 추가
model Payroll {
  id           String   @id
  tenantOrgId  String   // 본인 조직이면 self.id, 관리 조직이면 ManagedOrg.id
  employeeId   String
  amount       Decimal
  // ...
  @@index([tenantOrgId])
}

// 새 모델: 관리 조직
model ManagedOrg {
  id           String   @id
  ownerOrgId   String   // 누가 관리하는지 (= 결제 주체)
  name         String
  createdAt    DateTime
  status       ManagedOrgStatus  // ACTIVE / PAUSED / TERMINATED
  // ...
}

// 권한: ownerOrg의 컨설턴트는 모든 ManagedOrg 접근 가능 (ReBAC scope: tenant:*)
// 위탁 직원은 본인 ManagedOrg만 (ReBAC scope: tenant:)

④ 관리 조직 라이프사이클

/managed-orgs (관리자 페이지)

추가

조직명 + 사업자번호 + 위탁 범위(Pack 선택) → 결제 가산

위임 초대

고객사 직원에게 외부 포털 토큰 발급 → 자기 데이터 직접 입력

해지

30일 보관 → 데이터 export 또는 hard delete
